# Система мониторинга температуры МИВК ОИЯИ

> Прогнозирование скачков температуры в системе охлаждения с помощью XGBoost (изначально сравнивались различные архитектуры, для практической реализации остановились на XGBoost)

##  О проекте

Разработан прототип системы раннего предупреждения о скачках температуры.  
Система анализирует временные ряды показаний датчиков и предсказывает, произойдёт ли критическое повышение температуры в ближайшие 20 минут.

**Ключевые характеристики:**

- Бинарная классификация (скачок / норма)
- Горизонт прогноза: 20 минут
- Порог скачка: +10% относительно текущего значения
- Метрика качества: F-macro = 0.822

##  Быстрый старт (без контейнеризации)

### 1. Клонирование репозитория

git clone https://github.com/Doshchepkov/jinr-monitoring_final.git

cd jinr-monitoring_final

### 2. Установка зависимостей

pip install -r requirements.txt

### 3. Подготовка данных

- Для актуальных данных (только для системных администраторов МиВК) поместите исходные CSV-файлы в папку source_datasets/ и выполните:

python build_dataset.py 

- Для теста на исторических данных:

python datasets/download_dataset.py

### 3.1 Создание буфера положительных классов

Будет использоваться для борьбы с петлей обратной связи на инференсе (при дообучение подмешиваются положительные классы)

python src/create_buffer.py \
    --data-path datasets/merged_dataset2.csv \
    --output-dir buffer \
    --L 60 \
    --H 30 \
    --n 0.1 \
    --max-samples 1000

### 4. Визуализация данных

python visualize_data.py

Будут сгенерированы графики:

1) correlation_matrix.png — тепловая карта корреляций
2) walk_forward_validation.png — разбиение на train/val/test
3) *_positive_episode_*.png — примеры положительных эпизодов

### 5. Обучение модели

python train_model.py --data-path datasets/merged_dataset2.csv

После обучения:

Модель сохранится в models/final_xgb.pkl

Логи сохранятся в logs/

### 6. Интерпретация модели 

python interpret_model.py --model=models/final_xgb.pkl --data=datasets/merged_dataset2.csv

Графики интерпретации (SHAP):

- shap_summary_bar.png — ранжирование признаков
  
- shap_summary_beeswarm.png — распределение влияния признаков
  
-  shap_waterfall_example.png — пример для конкретного эпизода

### 7. Результаты тестирования

| Fold | Порог | Val F-macro | Test F-macro |
|------|-------|-------------|--------------|
| 1 | 0.10 | 0.912 | 0.723 |
| 2 | 0.10 | 0.696 | 0.560 |
| 3 | 0.30 | 0.927 | 0.924 |
| 4 | 0.20 | 0.956 | 0.957 |
| 5 | 0.40 | 0.964 | 0.945 |

**Средний Test F-macro: 0.822**

## Быстрый старт с контейнеризацией (дополнительно)

### 1. Сборка контейнера

docker-compose -f docker/docker-compose.yml build

### 2. Запуск API

docker-compose -f docker/docker-compose.yml up -d api

API будет доступен по адресу: http://localhost:8000/predict

Пример запроса через curl:

curl -X POST "http://localhost:8000/predict" \
  -H "Content-Type: application/json" \
  -d '{"data": [[...], [...], ...]]}'

### Прогнать все команды из основоного пайплайна внутри созданного контейнера (загрузка датасета, загрузка буффера, обучение модели)

### 3. Тестирование локального API

python test_api.py

Скрипт вернёт предсказание модели для тестового эпизода

Можно визуализировать входные данные и вероятности скачка

### 4. Дообучение через Airflow

В dags/retrain_dag.py описан DAG для автоматического дообучения модели.


Для запуска:

airflow db init
airflow scheduler
airflow webserver --port 8080

⚠️ Возможны конфликты при параллельном доступе к данным, особенно если DAG запускается одновременно с API.
